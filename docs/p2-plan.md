# P2 商业化执行计划（中国场景）

> **目标**：1 周内能从中国大陆用户收到第一笔付款，半月内做到自动收款 + 自动发码。
>
> **核心约束**：
> - 卖家（作者）在中国大陆 → 不能用 Stripe / PayPal 等海外商户
> - 买家在中国大陆 → 不能挂 Twitter / Product Hunt 等推广渠道
> - 没有营业执照 → 不能直连微信支付 / 支付宝官方
>
> **原则**：能用最 hacky 的方式先收到钱，再投资基础设施。**头 50 个付费用户全手动是 OK 的**——朋友圈 + 微信发码就能跑。

---

## 三阶段递进

每阶段都能独立收钱、独立运转。后一阶段是"自动化前一阶段的手工活"。

### Stage 0 — 朋友圈微信收款（Day 1-3）

**目标：能从朋友 + 朋友的朋友手里收到第一笔钱。**

- ❌ 没有 landing page
- ❌ 没有后端
- ❌ 没有用户账号
- ❌ 没有自动化

**怎么收钱**：
- 朋友圈发产品介绍 + 个人**微信收款码**截图
- 群里（数据 / 运营 / 独立开发者群）发同样的内容
- 用户付完钱微信截图给作者

**怎么发码**：
- 作者本地有 `npm run gen-license -- <email> <days>` CLI（已建好）
- 收到付款 → 跑命令生成 license → 微信发用户

**怎么宣传**：
- 朋友圈一条
- 即刻一条（数据 / 独立开发者圈，可能转发）
- V2EX「分享创造」节点一帖
- 微信群发（你自己在的相关群）

**优点**：0 后端、0 平台抽成、0 营业执照、立刻能收钱  
**缺点**：作者每个用户手动 ~3 分钟；秒级响应不可能；规模化不行

---

### Stage 1 — 落地页 + admin 半自动（Day 4-5）

**目标：路人能找到你，作者发码从 3 分钟 → 30 秒。**

**Landing page**：
- 单页中文 HTML，部署到 **Cloudflare Pages**（免费、免备案、国内访问凑合）
- 内容：一段产品介绍 + 价格 + 截图 / 动图 + 加微信入口（二维码）+ FAQ
- 域名：用 Cloudflare Registrar 买一个 `.com` / `.app` 即可，免备案
- 不接付款链接——继续走 Stage 0 微信收款流

**admin 半自动**：
- Cloudflare Workers + KV
- 一个 admin 页（HTTP Basic Auth 锁着）
- 输入邮箱 + 时长 → 生成 license → 一键复制
- 历史查询：按邮箱搜过往发过的 license

**KV schema**：
```
licenses/{licenseId}: { email, issuedAt, expiresAt, tier, revoked: false }
emails/{email}: [licenseId, ...]
```

**收款仍然手动**——作者看到付款 → 打开 admin → 30 秒发码 → 微信发用户。

---

### Stage 2 — 爱发电自动收款 + 自动发码（Day 6-7+）

**目标：用户付款 → 爱发电私信收到 license，0 人工。**

- 用 **爱发电（afdian.com）** 做收款渠道
  - 大陆个人开发者可注册，免营业执照
  - 支持微信支付 / 支付宝
  - 抽成 ~10%（可接受）
  - 有"自动发货 / 解锁奖励"功能
- 配置爱发电的「计划档位」：¥30 / 月 = 月度激活，¥288 / 年 = 年度激活
- 接 [爱发电开放 API](https://afdian.net/p/8f43b6406b2a11ea91c652540025c377) webhook：用户付款成功 → CF Workers 接收 → 调本地发码逻辑 → 通过爱发电私信回传 license
  - 备选：定时拉取订单接口
- Landing page 改"加微信购买"为"前往爱发电购买"

**风险**：爱发电的 webhook 文档需要实测，可能有坑。**Plan B**：保留 admin 页面手动发码作为兜底。

## 收款渠道选择

| 选项 | 准入门槛 | 抽成 | 推荐 |
|---|---|---|---|
| **个人微信 / 支付宝收款码** | 0 | 0 | 🟢 **Stage 0 首选** |
| **爱发电** | 个人实名 | ~10% | 🟢 **Stage 2 自动化** |
| 知识星球 | 实名 | ~10% | 🟡 备选（带社群） |
| 公众号付费内容 | 实名 + 公众号 | ~5% | 🟡 备选（公众号沉淀） |
| 微信小商店 | 营业执照（个体工商户也行） | ~6% | 🟡 体量起来再考虑 |
| 直连微信支付商户 | 营业执照 + 对公 | ~0.6% | 🔴 门槛高、不做 |
| Stripe / PayPal / Creem | 海外身份 / 银行 | ~5% | 🔴 大陆卖家注册难、不做 |

## 推广渠道

**主要阵地**（按优先级）：
1. **即刻** —— 互联网 / AI / 独立开发者圈，最对口的受众
2. **V2EX** —— 「分享创造」节点首发
3. **朋友圈 + 微信群** —— 第一波用户必经
4. **小红书** —— 录视频教程，覆盖运营 / 数据分析师
5. **B 站** —— demo 视频
6. **知乎** —— 找相关问题写长答（"小红书 / 抖店 数据怎么自动下载"等）
7. **掘金 / SegmentFault** —— 写技术文章覆盖开发者

**不做**：
- ❌ Twitter / X（墙了，无受众）
- ❌ Product Hunt / HackerNews（英文 + 中文用户极少）
- ❌ Reddit（墙了）
- ❌ LinkedIn（国内基本不用）
- ❌ 知识星球 / 付费社群拉新（太重，先验证产品）

---

## 防破解

**V1 不做**——规模化前是过度设计。

接受的风险：
- license 共享：一个用户多设备用 OK，看到批量滥用再加机器指纹
- license 伪造：HMAC secret 泄露能伪造，1000+ 用户时换公私钥方案

---

## 退款政策

- **14 天试用够决策**——付款后**不退**，购买页明确写
- 例外：明显技术问题（扩展完全不能用）退 50%
- 不接受"我以为它能 X"类退款

## 不做的（永远）

- 教程视频系列（一份 [docs/guide.md](guide.md) 够了；demo 视频是营销不是教程）
- 用户分群 / persona
- Affiliate 推广分成
- 企业版定价 / 销售线索表单
- 复杂的退款 / 客服流程
- 邮件营销（中国用户不看邮件，用微信 / 公众号）
- 多语言（先做中文。海外用户是 V2 命题，不在 V1 范围）
- Twitter / 海外社交媒体推广
