# webXport 使用指南

> Chrome 插件。在你已登录的浏览器里录一段操作，定时重放，把数据下到本地。
>
> 也能让 Claude Code / Cursor 这些 AI agent 直接调用，构成"AI 帮你抓数据 → 分析"的工作流。

---

## 它能干什么

- 你在小红书千帆 / 抖店 / 千牛 / 京麦 / 视频号助手 / 微信视频号 / 各平台广告后台，每天要点同样的几下导出报表 → 让 webXport 录一次，**定时帮你跑**
- 跑完文件落到 `~/Downloads/webxport/{脚本名}/{日期}/`，按业务日期归档
- AI agent（Claude Code / Cursor / Codex）可以通过 MCP 协议调用本插件——你跟 AI 说"看下昨天的小红书数据"，**AI 自己驱动浏览器抓数据 + 分析**，不用切窗口

## 它不是

- 不是独立爬虫——靠的就是你 Chrome 里现成的登录态，平台无法识别为机器流量
- 不会代替你登录 / 输入验证码——保持你已登录就行
- 没有内置 AI——AI 在调用方（Claude Code 等），插件只负责"拿数据"
- 没有官方脚本库——每个平台 UI 都不一样，每个用户要的报表也不一样，你自己录最准

## 安装

```bash
git clone <repo>
cd webXport
./install.sh
```

`install.sh` 会：
1. 构建 Chrome 扩展（产物在 `dist/`）
2. 构建 MCP server
3. 自动把 MCP 注册到 Claude Code（user scope，所有 VS Code window 都能看到）

脚本跑完会提示你完成最后一步——**手动加载扩展到 Chrome**：

1. 打开 `chrome://extensions/`
2. 右上角打开「开发者模式」
3. 点「加载已解压的扩展程序」→ 选 `dist/` 目录
4. Chrome 工具栏右上角会出现 **W 图标**

以后改了代码再跑一次 `./install.sh` 即可，Chrome 那边只要在扩展卡片上点一下 ↻ 重载。

---

## 5 分钟录第一个脚本

1. 在 Chrome 里登录你要抓的平台（例如 `ark.xiaohongshu.com`）
2. 点工具栏 W 图标 → popup 弹出 → 输入脚本名（如"小红书每日数据"）→ 点「开始录制」
3. 在页面上正常操作：点「数据中心」→ 选时间范围 → 点「导出」→ 等下载弹窗 → 点「下载」
4. 操作完点**页面右下角浮动条**的「停止录制」
5. 脚本就保存好了

**验证它能跑**：popup → 找到刚录的脚本 → 点「立刻运行」。Chrome 会自动重复你刚才的操作，几秒后文件落到 `~/Downloads/webxport/小红书每日数据/{今天日期}/`。

> ⚠️ 录制时**只录主动操作**（点击 / 输入 / 选择）。鼠标悬停、滚动、移动这些不会被录。这是有意的设计——录中间态会让重放变得脆弱。

## 设置定时

popup → 选脚本 → 「编辑」→ 设置「时间」（`HH:mm` 24 小时制）→ 保存。

每天到那个点会自动跑一次。

> ⚠️ **Chrome 必须开着才会跑**。webXport 不会唤醒电脑或自启动 Chrome。如果你睡前关了电脑，第二天早上的定时会跳过这次（**不补跑**）。要它每天跑就别关 Chrome。

## 文件去哪了

固定结构，跨时区不会错：

```
~/Downloads/
  webxport/
    小红书每日数据/
      2026-05-09/
        note-data.xlsx
      2026-05-10/
        note-data.xlsx
    抖店订单/
      2026-05-10/
        orders.csv
```

按业务日期归档（从下载文件名提取），不用 `new Date()`——跨日触发不会归错档。

## 让 AI 调用（高阶）

如果你装了 Claude Code 或 Cursor，`install.sh` 已经把 MCP 注册好了。

在 Claude Code（任意 VS Code window 都行）直接说：

> 列出我的 webxport 脚本

或更进阶：

> 用 webxport 跑一下"小红书每日数据"，跑完帮我看互动率有没有掉

AI 会**自己调用 webxport** → 等数据下载好 → 读 Excel → 给结论。你不用切窗口、不用打开 Excel、不用复制粘贴。

> 💡 **第一次用前先点工具栏 W 图标弹一下 popup**——唤醒插件后台。Chrome MV3 限制后台 30 秒不活跃就休眠，弹一下 popup 就重连了。AI 调挂了再点一下就行。

可用的 MCP 工具（不用记，AI 会自己用）：

| 工具 | 作用 |
|---|---|
| `list_scripts` | 列出所有脚本 + 最近一次运行摘要 |
| `run_script(name, waitForResult=true)` | 跑指定脚本，默认等 drain 完成返回文件路径 |
| `get_runs(name?)` | 单个或全部脚本的运行历史（最近 10 次） |
| `get_run_status` | 当前是不是有脚本在跑、跑到第几步 |

## 常见问题

**Q. 脚本跑挂了怎么办？**

popup 里看「运行历史」，会写"第 N 步失败：xxx"。常见原因：

- session 过期 → 在浏览器重新登录一次再跑
- 平台 UI 改版（按钮位置变了 / 重命名了）→ 删掉脚本重录一次（5 分钟的事）
- 弹窗 / 验证码挡住了 → 处理掉再跑

不会自动恢复——猜错代价比让你重录大。

**Q. 我录的时候点错了一步**

那次录制放弃，重新开始。脚本要"干净"，多余步骤会让重放变脆。

**Q. 我半夜电脑没开，定时漏了一天**

不会补跑。早上手动点「立刻运行」一次。

**Q. 多个 Claude Code 窗口同时用？**

可以。每个 session 自己跟插件通信，最多 6 个并发。

**Q. 在 Cursor / Codex / 其他 AI agent 里能用吗？**

只要支持 MCP 协议都能用。Cursor 自动从 `~/.claude.json` 读，其他 agent 看各自文档。手动注册命令：

```bash
claude mcp add -s user webxport node /path/to/webXport/mcp-server/dist/index.js
```

## 限制 / 注意

- **不可逆动作（删除 / 支付 / 确认提交 / 退款 / 解绑 / 注销）会被自动拦截**——硬编码在 `src/shared/safety.ts`，不可绕过。这是产品红线，不是 bug
- 文件只能落在 `~/Downloads/` 子目录下（Chrome 扩展 API 限制）。想去 Desktop / 别的地方自己手动移或写个脚本
- 平台改版没人通知你，发现不工作了**重录一次**就行——别尝试改脚本里的 selector，会越改越乱
