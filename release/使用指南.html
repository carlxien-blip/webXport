webXport 使用指南

Chrome 插件——在你已登录的浏览器里录制一次操作，定时重放下载数据报表。也能让 AI agent（Claude Code / Cursor）通过 MCP 直接调用，实现「AI 帮你抓数据 + 分析」闭环。

🚀 5 分钟跑通

第一步：安装扩展

  1. 解压旁边的 webxport-extension/ 文件夹（如果还没解压）
  2. 打开 Chrome → 地址栏访问 chrome://extensions/
  3. 右上角打开「开发者模式」开关
  4. 点「加载已解压的扩展程序」→ 选 webxport-extension/ 文件夹
  5. Chrome 工具栏右上角出现 W 图标
⚠️ Chrome 顶部会有一条黄色警告"关闭开发者模式"。这是因为目前还在 Chrome Web Store 审核中（预计 1-2 周通过）。通过之后切换成商店版，警告条消失。

第二步：录第一个脚本

  1. 在你已登录的平台后台打开（小红书千帆 ark.xiaohongshu.com、抖店、千牛、京麦、视频号助手等）
  2. 点工具栏 W 图标 → popup 弹出 → 输入脚本名（如"小红书每日数据"）→ 点「开始录制」
  3. 在页面上正常操作：点「数据中心」→ 选时间范围 → 点「导出」→ 等下载弹窗 → 点「下载」
  4. 操作完点页面右下角浮动条的「停止录制」
  5. 脚本保存好了 ✓

验证：popup 里点该脚本「立即运行」，Chrome 会自动重复你刚才的操作，几秒后文件落到 ~/Downloads/webxport/小红书每日数据/{今天日期}/。

第三步：设置定时

popup → 选脚本 → 「编辑」 → 设置「时间」（HH:mm 24 小时制）→ 保存。每天到点自动跑。

⚠️ Chrome 必须开着才会触发定时。半夜电脑关机就会跳过那次（不补跑）。要每天准时跑就别关 Chrome。

❓ 常见问题

脚本跑挂了怎么办？

popup 里看「运行历史」，会显示"第 N 步失败：xxx"。常见原因：

  • session 过期 → 浏览器里重新登录一次再跑
  • 平台 UI 改版 → 删掉脚本重录一次（5 分钟的事）
  • 弹窗 / 验证码挡住了 → 手动处理掉再跑

我录的时候点错了一步

那次录制放弃，重新开始。脚本要"干净"，多余步骤会让重放变脆。

我半夜电脑没开，定时漏了一天

不补跑。早上手动点「立即运行」一次。

多台电脑能用同一个 license 吗？

暂时不限制（自己家 + 公司电脑这种场景 OK）。批量分享发现会上设备绑定。

退款？

14 天试用足够判断了。付款后不退（明显的技术故障除外）。

担心数据安全？

所有数据本地存储（chrome.storage.local + Downloads/）。不上传任何服务器、不收集邮箱手机号、不读取 cookies。详见 隐私政策。
